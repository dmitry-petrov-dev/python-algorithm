Реализованы 3 варианта кода для поиска максимального элемента из минимальных элементов столбцов матриц.
Вариант 1. В качестве матрицы используется список, выделена отдельно переменная для поиска максимального значения.
В результате количество занятой памяти получилось 2116.
Вариант 2. Второй вариант также использует список для хранения матрицы, но для поиска максимального значения не
выделяется отдельно переменная. Вместо этого делается сортировка списка и берется первый элемент. В результате
количество занятой памяти получилось 2088.
Вариант 3. Для хранения матрицы используется кортеж, вместо списка. В результате количество занятой памяти уменьшилось и
занимает 2048.

Вывод:
Для хранения неизменяемых объектов лучше использовать кортежи вместо списков, так как кортежи занимают меньше памяти.
